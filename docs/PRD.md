# 任務管理系統 PRD (產品需求文件)

## 1. 專案概述
- **背景與動機**：現代人生活與工作節奏緊湊，往往需要一個簡潔有效的工具來記錄並追蹤待辦事項，避免遺漏重要工作。為了提高個人生產力與時間管理效率，開發此「任務管理系統」。
- **目標用戶**：所有人（包含學生、上班族、自由工作者等需要管理日常雜務與工作任務的人）。
- **核心價值主張**：提供一個直覺、易用的介面，讓使用者能快速新增與管理任務，並給出待辦任務的提醒，減少使用者的大腦記憶負擔。

## 2. 功能需求
系統核心功能如下，以使用者故事（User Story）呈現：

1. **新增任務**
   - 作為 **使用者**，我希望 **能夠快速新增一個任務（包含標題、預計完成時間與詳細描述）**，以便 **記錄我需要處理的事情**。
2. **編輯任務**
   - 作為 **使用者**，我希望 **能夠修改現有任務的內容與細節**，以便 **在任務需求變更時能保持最新資訊**。
3. **刪除任務**
   - 作為 **使用者**，我希望 **能夠刪除錯誤建立或不再需要的任務**，以便 **保持任務列表的整潔**。
4. **標記完成狀態**
   - 作為 **使用者**，我希望 **能夠將已做完的任務標記為「完成」**，以便 **清楚分辨尚未處理的任務，並獲得成就感**。
5. **提醒待完成任務**
   - 作為 **使用者**，我希望 **系統能列出或提醒尚未完成及即將到期的任務**，以便 **我不會錯過任何重要事項**。

## 3. 非功能需求
- **技術限制**：
  - 後端框架：Flask
  - 模板引擎：Jinja2
  - 資料庫：SQLite
- **效能考量**：
  - 網頁操作的回應時間應小於 1 秒。
  - 操作體驗必須平滑，確保清單顯示過長時系統仍有良好效能。
- **安全考量**：
  - 避免常見的 Web 攻擊，如：使用 Jinja2 預設的特殊字元跳脫機制防範 XSS。
  - 預防 SQL Injection，確保從前端接進來的資料需經過過濾後再進資料庫。

## 4. MVP 範圍 (最小可行產品)
- **Must Have (必須要有)**：
  - 基本任務 CRUD：新增、檢視清單、編輯、刪除任務。
  - 狀態切換：任務可標記為「已完成」或「未完成」。
- **Should Have (應該要有)**：
  - 任務到期時間與系統高亮/置頂提示（達成提醒功能）。
  - 對任務進行簡單排序（如依建立時間或預計完成時間）。
- **Nice to Have (加分項目)**：
  - 多使用者帳號登入與個人化任務切割。
  - 分類或標籤系統。
  - 任務搜尋與過濾功能。
