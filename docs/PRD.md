# 任務管理系統 PRD

## 1. 專案概述

- **背景與動機**：為了解決個人日常待辦事項的追蹤問題，需要一個簡單直觀的介面來記錄與管理任務，避免遺漏重要事項。
- **目標用戶**：我自己（單一個人使用者）。
- **核心價值主張**：提供輕量化、操作直覺的介面，快速完成任務的新增、追蹤與狀態更新，提升個人生產力。

## 2. 功能需求

以下為系統的主要功能與對應的使用者故事（User Stories）：

1. **新增任務**
   - 作為 **我自己**，我希望 **能夠輸入任務標題與內容並新增任務**，以便 **記錄當下需要處理的待辦事項**。
2. **編輯任務**
   - 作為 **我自己**，我希望 **能夠修改已存在任務的內容與標題**，以便 **在任務細節發生變化時能夠隨時更新**。
3. **刪除任務**
   - 作為 **我自己**，我希望 **能夠刪除錯誤輸入或不再需要處理的任務**，以便 **保持任務清單的簡潔**。
4. **標記任務完成狀態**
   - 作為 **我自己**，我希望 **能夠自由切換任務的「未完成/已完成」狀態**，以便 **清楚掌握哪些事情已經做完、哪些還需要努力**。
5. **顯示待完成任務**
   - 作為 **我自己**，我希望 **能夠在畫面上清楚看到所有未完成的任務列表**，以便 **專注於接下來優先需要處理的工作**。

## 3. 非功能需求

- **技術限制**：依據團隊標準（Skill 約定），將使用 **Flask + Jinja2 + SQLite** 進行全端與資料庫開發。
- **效能考量**：作為單人用的輕量化系統，首頁與任務操作的響應時間需盡可能快速，不應有明顯延遲。
- **安全考量**：由 Flask/SQLite 實作，需避免基本的 SQL Injection（建議使用 ORM 或 Parameterized Query，如 SQLAlchemy 或純 SQLite 參數綁定），表單應有基本的 CSRF 防護（若需未來擴展）。

## 4. MVP 範圍

為了快速驗證並讓系統盡快上線，功能劃分如下：

- **Must Have (絕對要有)**
  - 任務列表展示（首頁能看到待完成任務）
  - 新增任務表格
  - 能夠修改與刪除任務
  - 任務打勾/標記完成
- **Should Have (最好要有)**
  - 列表排序（例如：最新新增的在最上面，或未完成的在最上面）
  - 基礎 RWD 設計（如果我想在手機瀏覽器上查看也能正常操作）
- **Nice to Have (有會更好，可未來擴充)**
  - 任務分類或標籤功能
  - 任務期限與過期提醒
